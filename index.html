What you'll be able to do

Outcomes that compound across every GenAI project you ship.

The course is built around three durable competencies. Every module's labs ladder up to one of them — so by the end you can stand up an evaluation pipeline on a new project in a single afternoon.

01 · Measure

Treat quality as a distribution, not a pass/fail.

Set up datasets, judges, and code-based scorers that produce stable, statistically meaningful signals across thousands of runs.

02 · Trust

Align automated judges with human standards.

Collect expert feedback, calibrate make_judge(), and gate releases only when the judges actually agree with your reviewers.

03 · Operate

Run eval continuously, governed, and on budget.

AI Gateway, Inference Tables, registered scorers, and Unity Catalog — wired together as one production-grade LLMOps stack on Databricks.

Learning path

Three phases — Foundation, Build, Ship.

The modules cluster into three phases. Each phase culminates in a working artifact you can carry forward — a versioned UC dataset, a custom judge suite, and a production-ready evaluation pipeline.

Phase 01 · Foundation

Get the harness running.

  • Modules 01–02 — concepts & datasets
  • Stand up MLflow 3 on Databricks
  • First UC evaluation dataset, versioned
  • Baseline run with built-in judges
Phase 02 · Build

Author scorers that fit your domain.

  • Modules 03–05 — predict_fn, judges, tracing
  • Wire any agent shape into evaluate()
  • Compose code + LLM-based scorers
  • RAG & agent evaluation via traces
Phase 03 · Ship

Operate it in production.

  • Modules 06–08 — gateway, alignment, governance
  • AI Gateway guardrails + Inference Tables
  • Align judges to human reviewers
  • Cost, anti-patterns, deployment gates
Who this is for
